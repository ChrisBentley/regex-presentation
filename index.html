Getting started:

'/' denotes the start and end of a regex

The following regex /abc/ will match the string 'abcdef' like so 'abcdef'

Adding a g after the end of the '/' adds a global flag to the regex

This means that it will continue to match after it's found the first iteration of a matching string.

e.g. /abc/g matches the string 'abcdef abcd' like so: 'abcdef abcd'

Matches for types:

. matches any character.

e.g. '/./' matches 'abc' and '%5£' like so 'abc' '%5£'

If you need to match a period you can escape the character in the regex using a blackslash i.e. /\./ would match '.'

\d matches any digit.

e.g. '/\d/' matches '123' like so '123'

It will not match 'a54' -> 'a54'

\w matches any alphanumerical character.

e.g. '/\w/' matches '123' and 'a54' like so '123' 'a54'

It will not match '%a5' -> '%a5'

\s matches any whitespace character.

e.g. '/\s/' matches ' 23' like so '_23'

Capital letter version of these type matchers will do the opposite. i.e.

\W matches any non-alphanumerical character.

\S matches any non-whitespace character.

Match multiple letters in a certain position using [ ]

If I have the strings 'cat', 'mat' and 'bat'

But I only want to match 'mat' and 'bat'

I would use the regex /[mb]at/

Matches like so: 'cat' 'mat''bat'

Because the first letter of cat does not match the regex, the string will not be matched.

We can also match ranges of letters or numbers using a hyphen between them like so:

  • [A-Z]
  • [a-z]
  • [0-9]

The NOT operator inside of these square brackets is the carot ^

  • e.g. '[^a]' means not an 'a' character

If you want to match characters that are repeated multiple times you can use { }

e.g. /WE{6}!/ matches 'WEEEEEE!'

We can also match a range of repeated characters using {m,n}

e.g. hello{1,4}! matches 'hello!' and 'helloooo!'

If we want to keep matching any number of repeated characters we can use:

* for 0 or more repetitions

+ for 1 or more repetitions.

e.g. yus+ or yus* match 'yussssss...'

? follows an optional character

e.g. /woot?/ would match 'woo' and 'woot'

^ and $ denote the start and end of the thing to match.

This is especially useful is you need to match exact things.

e.g. If you have the string 'API Call: successful' you might be tempted to use the regex /successful/

But what if your logs also contain 'Error: Expected "API Call: successful"'?!

Anchors to the rescue! We would use: /^API Call: successful$/

( ) are used to capture parts of a message

e.g. If we have the strings of the format '5 Jan 1985' you could capture the year using:

/\d+ \s [A-z]+ \s ([0-9]+)/
